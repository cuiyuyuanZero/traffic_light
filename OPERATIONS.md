# Traffic Light Service 操作手册

本工具是一个实时的状态监控悬浮窗，专门用于追踪 **Codex** 和 **Claude** 的工作状态，通过信号灯颜色直观展示它们是否正在思考、执行工具或处于空闲状态。

## 🚀 快速启动

在项目根目录下执行：
```bash
npm run start
```
执行后，桌面上会出现一个半透明的圆形信号灯悬浮窗。

## 🚦 信号灯说明

| 颜色 | 状态 | 含义 |
| :--- | :--- | :--- |
| 🟢 **绿色** | **Idle (空闲)** | Agent 已完成所有任务，正在等待你的下一条指令。 |
| 🔴 **红色** | **Thinking (思考)** | Agent 正在进行深度推理、生成回复或准备上下文。 |
| 🟠 **橙色** | **Executing (执行)** | Agent 正在操作工具（如读取文件、执行 Shell 命令、搜索网页）。 |

## 🛠️ 核心功能

1.  **100% 精准状态识别**：通过直接解析 Codex 的 Rollout JSONL 日志，实现毫秒级状态同步。
2.  **热启动 (Warm-up)**：启动即扫描历史日志，即便在任务进行中开启监控，也能立刻找回当前真实状态。
3.  **防抖处理**：智能识别“任务结束”信号，防止在多轮对话间产生不必要的闪烁。
4.  **全平台支持**：支持监控文件读取、代码补丁应用、网页搜索及上下文自动压缩等后台事件。

## 📂 项目结构

- `server.js`: 后端核心逻辑，负责日志追踪、状态判定及 WebSocket 广播。
- `main.js`: Electron 入口，管理悬浮窗的展示与样式。
- `public/`: 前端界面，包含信号灯的 UI 和动画逻辑。
- `notify_hook.js`: (可选) 用于通过 Webhook 手动触发状态变更。

## ❓ 常见问题排查

- **悬浮窗不显示**：
  - 检查端口 `19001` 是否被占用。
  - 确保已运行 `npm install` 安装依赖。
- **状态长时间不更新**：
  - 信号灯具有 5 分钟的自动超时恢复机制，如果 Agent 彻底卡死，它会自动变回绿色。
  - 尝试重启服务：关闭窗口后重新运行 `npm run start`。
- **日志路径配置**：
  - 默认自动检测 macOS 标准日志路径。如果路径不一致，请在 `server.js` 中修改 `getLatestCodexRollout` 等函数。
